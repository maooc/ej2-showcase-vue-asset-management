## 2. Vue + Syncfusion 组件集成机制

### 2.1 Syncfusion 组件使用方式

项目使用 **Syncfusion Essential JS 2** 组件库，采用 Vue 3 Composition API 风格集成：

#### 核心依赖组件
```json

  "@syncfusion/ej2-vue-buttons": "*",      // 按钮、单选按钮
  "@syncfusion/ej2-vue-calendars": "*",    // 日期选择器、日期范围选择器
  "@syncfusion/ej2-vue-charts": "*",       // 图表组件
  "@syncfusion/ej2-vue-dropdowns": "*",    // 下拉框、自动完成
  "@syncfusion/ej2-vue-inputs": "*",       // 输入框、数字输入、上传组件
  "@syncfusion/ej2-vue-lists": "*",        // 列表视图
  "@syncfusion/ej2-vue-grids": "*",        // 数据表格
  "@syncfusion/ej2-vue-navigations": "*",  // 侧边栏、标签页
  "@syncfusion/ej2-vue-notifications": "*",// Toast 通知
  "@syncfusion/ej2-vue-popups": "*",       // 对话框、工具提示
  "@syncfusion/ej2-vue-treemap": "*"       // 树图
```

#### 审批/拒绝请求（已实现）
```javascript
// AllRequests.vue - 第107-126行
function approveClick(args) {
    // 1. 获取当前用户信息
    var userName = new DataManager(employeeData)
        .executeLocal(new Query().where(new Predicate('id', 'equal', store.state.currentUserID)))[0]['Employee'];
    var userImg = new DataManager(employeeData)
        .executeLocal(new Query().where(new Predicate('id', 'equal', store.state.currentUserID)))[0]['ImgSrc'];
    
    // 2. 获取行数据
    let rowObj = gridObj.value.ej2Instances.getRowObjectFromUID(target.parentElement.parentElement.getAttribute('data-uid'));
    var row = rowObj.data;
    
    // 3. 更新请求状态
    var grid = document.querySelector('.e-grid')['ej2_instances'][0];
    var index = parseInt(args.target.closest('td').getAttribute('index'));
    grid.editModule.updateRow(index, {'TaskID': row['TaskID'], 'Status': 'Approved'});
    
    // 4. 添加活动记录
    var activity = {
        'id': store.state.activityData.length,
        'Employee': userName,
        'ImgSrc': userImg,
        'TimeStamp': new Date().toLocaleDateString(),
        'Message': 'A new request for ' + row['RequestedItem'] + ' has been approved'
    };
    store.dispatch('addActivity', Object.assign({}, activity));
    
    // 5. 显示成功提示
    root.root.refs.toastRef.show({
        title: 'Successs!',
        content: 'Request approved successfully',
        cssClass: 'e-toast-success',
        icon: 'e-success toast-icons'
    });
    
    // 6. 更新待处理请求数量
    updatePendingRequests();
}
```

#### Actions 设计（异步操作）

```javascript
// main.js - 唯一的 Action
actions: {
    addActivity(context, activity) {
        return new Promise((resolve, reject) => {
            // 验证用户是否存在
            var userExist = context.state.empInput.find((user) => {
                return user.Employee === activity.Employee;
            });
            
            if (!userExist) {
                reject(new Error('No such user found!'));
            } else if (activity.Employee === '') {
                reject(new Error('User name cannot be empty!'));
            } else {
                context.commit('addActivity', activity);
                resolve(Object.assign({}, userExist));
            }
        });
    }
}
```

**说明：** 该 Action 虽然返回 Promise，但实际是同步验证逻辑，主要用于表单验证和错误处理。

### 5.2 组件拆分合理性评估

#### 优点
- ✅ 图表组件独立拆分（dashboard/*.vue），职责单一
- ✅ 对话框组件与列表组件分离，符合单一职责原则
- ✅ 活动模板组件（ActivityTemplate.vue）在 Dashboard 和 Notification 中复用
- ✅ 使用 Composition API，代码组织清晰

#### 不足
- ❌ 缺少通用的 Grid 基础组件，4 个列表页面存在大量重复代码
- ❌ 缺少通用的 Dialog 表单组件，4 个对话框组件结构相似
- ❌ 缺少 API 服务层封装（当前无真实后端接口）
- ❌ 缺少通用的日期筛选组件

### 5.3 设计模式应用

| 模式 | 应用场景 | 评价 |
|------|---------|------|
| **组件组合** | Dashboard 组合多个图表组件 | ✅ 良好 |
| **Props/Events** | 父子组件通信（对话框与列表页） | ✅ 标准 |
| **Vuex 状态管理** | 全局状态共享（活动记录、用户信息） | ✅ 合理 |
| **模板模式** | ListView 使用 ActivityTemplate | ✅ 良好 |
| **Provider/Inject** | updatePendingRequests 跨组件传递 | ✅ 良好 |

## 总结

本报告基于项目实际源码进行了深入分析，主要发现：

1. **数据管理**：业务数据来自 `datasource.js` 静态文件和 Vuex Store，未封装统一的 Axios/Fetch 业务接口层。唯一存在的异步请求是 AddHardware.vue 中的文件上传功能。

2. **已实现功能**：Grid 列表展示、列筛选、日期范围筛选、分页、编辑、删除、请求审批、活动记录、文件上传。

3. **未实现功能**：关键字搜索模块、独立的详情查看页面/弹窗、真实后端接口封装。

4. **代码质量**：存在较多重复代码，建议通过 Composables 和通用组件抽取来优化。

5. **架构设计**：组件拆分合理，图表组件独立，但缺少通用的 Grid 和 Dialog 基础组件。
